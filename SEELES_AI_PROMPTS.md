## 🎯 PROMPT 1: IDLE ANIMATION (8-DIRECTIONAL)

Copy this exactly into seeles.ai:

```
Create an 8-directional pixel art sprite sheet for a fantasy knight in idle pose.

CHARACTER DESCRIPTION:
- Medieval knight with glowing magical armor
- Cyan/blue glowing effects (magical aura, runes, trim)
- Gold/orange metallic accents and details
- White cross symbol on chest/center
- Standing in neutral idle stance

SPRITE SHEET FORMAT:
- Layout: 8 frames wide × 2 frames tall = 512×128 pixels total
- Each frame: 64×64 pixels
- Frame order (left to right): N, NE, E, SE, S, SW, W, NW
- Row 1: Idle pose frame 1 (neutral standing)
- Row 2: Idle pose frame 2 (subtle breathing/weight shift)

DIRECTION REQUIREMENTS:
- N (North): Character facing up-left (isometric north)
- NE: Facing up-right diagonal
- E (East): Facing right
- SE: Facing down-right diagonal
- S (South): Facing down-right (isometric south)
- SW: Facing down-left diagonal
- W (West): Facing left
- NW: Facing up-left diagonal

ANIMATION:
- Subtle idle animation (breathing, slight weight shift)
- Both frames should show same pose with minimal variation
- NO walking or action poses
- NO weapons drawn
- Ready/neutral stance

STYLE:
- Pixel art (isometric game style)
- Retro RPG aesthetic
- Clear silhouette
- Color scheme: Blues, cyans, golds, blacks, grays
- Transparent background
- Shadow beneath character (optional)

OUTPUT REQUIREMENTS:
- Format: PNG with transparency (or equivalent)
- Resolution: 512×128 pixels exactly
- Background: Transparent
- Character centered in each 64×64 frame
- Point filter (crisp pixels, no anti-alias smoothing)
```

## 🎯 PROMPT 2: WALK ANIMATION (8-DIRECTIONAL)

```
Create an 8-directional pixel art sprite sheet for a fantasy knight walking.

CHARACTER DESCRIPTION:
- Medieval knight with glowing magical armor (same as idle)
- Cyan/blue glowing effects matching idle animation
- Gold/orange metallic accents
- White cross symbol on chest
- Walking in forward motion

SPRITE SHEET FORMAT:
- Layout: 8 frames wide × 2 frames tall = 512×128 pixels total
- Each frame: 64×64 pixels
- Frame order (left to right): N, NE, E, SE, S, SW, W, NW
- Row 1: Walk cycle frame 1 (left leg forward, right leg back)
- Row 2: Walk cycle frame 2 (right leg forward, left leg back)

DIRECTION REQUIREMENTS:
- N (North): Walking facing up-left
- NE: Walking up-right diagonal
- E (East): Walking right
- SE: Walking down-right diagonal
- S (South): Walking down-right
- SW: Walking down-left diagonal
- W (West): Walking left
- NW: Walking up-left diagonal

ANIMATION:
- 2-frame walk cycle (legs swinging)
- Frame 1: One leg forward, smooth stride
- Frame 2: Other leg forward, opposite stride
- Continuous, looping walk motion
- Medium pace (comfortable movement speed)
- Upper body stays relatively stable

STYLE:
- Same pixel art style as idle animation
- Matches idle character design exactly
- Colors: Blues, cyans, golds, blacks, grays
- Transparent background
- Isometric game style

OUTPUT REQUIREMENTS:
- Format: PNG with transparency
- Resolution: 512×128 pixels exactly
- Background: Transparent
- Character centered in each 64×64 frame
- Smooth walk cycle (no jerky transitions)
- Point filter (crisp pixels)
```

## ✅ INTEGRATION CHECKLIST

Once you have the sprite sheets from seeles.ai:

1. **Download PNG files** (with transparency)
2. **Place in project:**
   ```
   Assets/Resources/Characters/Player/AnimationSprites/
   ├── idle.png          (512×128)
   ├── walk.png          (512×128)
   └── attack.png        (512×64 optional)
   ```
3. **Run PlayerSetupWindow:**
   - Tools > Iso World > Player Setup Wizard
   - Select idle + walk sheets
   - Click "Setup Everything"
4. **Test in-game!**

---

## 💡 TIPS FOR BEST RESULTS

- Be specific about the **8 directions** (most generators need this clarity)
- Ask for **pixel art style** explicitly (some default to realistic)
- Request **512×128 format** in the prompt (prevents resizing)
- Specify **transparent background** (essential for isometric)
- Ask for **consistent character** across all directions (same armor, colors)
- Request **point filter** / **crisp pixels** (not smooth anti-alias)
