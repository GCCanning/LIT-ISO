# Asset Forge Local Dashboard Spec

This is the near-term product shape for the local Sprixen-style tool. It is a clean-room local workflow for LIT-ISO assets, not a clone of Sprixen code, branding, prompts, or assets.

## Workspaces

- Generate: prompt/reference to candidates for character, mob, prop, item, tile, effect.
- Review: contact sheets, per-asset QA, approve/reject/needs-edit decisions.
- Cleanup: background removal, pixel snap, palette cap, nearest-neighbor resize, anchor normalization.
- Directions: 4D first, then 8D direction sheets.
- Animate: idle/walk/run/attack/cast/hurt/death frame sets.
- Tiles: terrain-only tiles, transitions, raised faces, biome packs.
- Props: trees, bushes, rocks, logs, stumps, boulders, long grass, shrubs, workstations.
- Items: 32/64/128 icon variants with transparent background.
- Effects: slash, hit spark, magic cast, projectile, harvest puff, death puff.
- Dataset: approved/rejected capture, captions, metadata, provenance.
- LoRA Lab: start/pause/resume/status/evaluate model training.
- Unity Export: generated handoff folders, manifests, import status.

## Data Contracts

### review_report.json

Generated by pack QA. It is authoritative for promotion readiness.

- `pack_name`
- `generated_utc`
- `terrain_count`
- `decoration_count`
- `pass_count`
- `review_count`
- `quality_contract`
- `items[]`

Each item:

- `name`
- `path`
- `category`: `terrain`, `decoration`, later `item`, `mob`, `character`, `effect`
- `width`
- `height`
- `opaque_pixels`
- `bbox`
- `has_meta`
- `status`: `pass` or `review`
- `issues[]`

### review_decisions.json

Human/tool review state. This must never override QA silently.

- `decision`: `pending`, `approved`, `rejected`, `needs_edit`
- `approval_blocked`: true when QA issues exist
- `training_capture`: true only for approved assets unless rejected examples are explicitly included
- `notes`

### approval_manifest.json

Records what was copied into generated handoff folders.

- `copied_count`
- `skipped_count`
- `failed_count`
- `copied[]`
- `skipped[]`
- `failed[]`

### dataset metadata.jsonl

Each line is a LoRA training record:

```json
{"file_name":"images/Forest_forest_grass_base.png","text":"LIT-ISO isometric terrain tile, Forest biome, forest grass base, pixel art, transparent background","source_path":"Assets/Generated/_Review/CodexBiomeStarter/Forest/forest_grass_base.png","decision":"approved","category":"terrain","biome":"Forest","pack":"CodexBiomeStarter"}
```

## LoRA Controls

The training loop must be laptop-friendly:

- `pause.request`: save a checkpoint and exit at the next step boundary.
- `stop.request`: save a stopped checkpoint and exit.
- `status.json`: current state, step, max steps, latest checkpoint, device, output folder.
- resume uses latest checkpoint plus step parsed from the checkpoint filename when available.

Commands:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File Tools\LoRA\start_resumable_litiso_training.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File Tools\LoRA\pause_litiso_training.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File Tools\LoRA\status_litiso_training.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File Tools\LoRA\start_resumable_litiso_training.ps1 -ResumeLatest
```

## Current Acceptance

- Review pack can be generated and QA-scored.
- Approval is fail-closed against QA issues.
- Approved assets copy only into generated handoff folders.
- Dataset capture can create LoRA-ready image/caption/metadata records.
- LoRA training has start, pause, status, and resume scripts.
- Pause/resume has been validated on `litiso_tile_prop_v1`: paused at step 375, resumed from the paused checkpoint, and continued past step 400.
